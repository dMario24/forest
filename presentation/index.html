서론: 패러다임의 전환

  • 배경: 기후 변화로 인한 전 지구적 산불 발생 빈도 및 강도 증가 (사회적, 경제적 재난 확대).
  • 변화: 경험과 인력 중심의 전통적 진화 체계 → 데이터 기반의 과학적 분석과 예측 시스템으로 전환.
  • 핵심 자원: 산림청, 기상청 등 공공기관의 방대한 산림/기상 데이터.
  • 목표: 산불 발생 메커니즘 규명, 확산 예측, 진화 자원 최적 배치.

  • 접근성/편의성: 설치 없이 구글 계정으로 즉시 사용 가능 (IT 비전공자 친화적).
  • 협업/공유: 구글 드라이브 연동, 코드 실시간 공유 (공동 연구에 최적).
  • 고성능 자원: GPU 무료 지원 (딥러닝, 위성 이미지 처리 등 고부하 작업 가능).

1.1.1 Colab 인터페이스 및 워크플로우

  • 파일 생성: `.ipynb` 확장자 (코드 + 텍스트 + 시각화 통합 문서).
  • 코드 셀 (Code Cell): 실제 파이썬 명령어 실행 공간.
  • 텍스트 셀 (Text Cell): 연구 방법론, 가설, 해석 기록 (연구 재현성 확보).

1.2 연구자를 위한 파이썬 핵심 문법

산불 데이터 유형에 맞춘 변수 및 데이터 타입 교육.

  • 수치형: 기온(Float), 습도(Float), 풍속(Float), 발생 건수(Integer).
  • 문자열: 지역명('강원도', 'Bejaia'), 요일('mon'). *인코딩(UTF-8, CP949) 주의 필요.
  • 리스트: 시계열 데이터 (일별 최고 기온 나열).
  • 딕셔너리: 구조화된 데이터 (특정 날짜의 기상 정보 Key-Value 쌍).

1.2.2 제어문과 라이브러리 활용

논리적 판단을 코드로 구현하는 실습.


# 예제: 산불 경보 발령 로직
humidity = 25  # 습도 (%)
wind_speed = 12 # 풍속 (m/s)

if humidity < 30 and wind_speed >= 10:
    print("🚨 [위험] 산불경보 발령 조건 충족")
else:
    print("✅ 기상 조건 양호")

필수 라이브러리: NumPy (수치 연산), Pandas (데이터 프레임 처리).

2.1 글로벌 표준 데이터셋: UCI 산불 데이터

2.1.1 알제리 산불 데이터셋 (Algerian Forest Fires)

  • 기간/장소: 2012년 6~9월, Bejaia & Sidi-Bel Abbes 지역 (244건).
  • 특징: 결측치가 거의 없는 깨끗한 데이터셋 (초보자용).
  • Target: `Classes` (fire / not fire) → 이진 분류 모델 실습 최적화.
  • FWI 요소: FFMC, DMC, DC, ISI, BUI, FWI (산불 기상 지수).

[표 1] 알제리 산불 데이터셋 주요 변수 정의

변수명전체 이름설명
TempTemperature정오 기준 최고 기온 (°C). 연료 건조의 주요 요인.
RHRelative Humidity상대습도 (%). 산불 위험과 반비례.
WsWind Speed풍속 (km/h). 확산 속도(ISI) 결정 변수.
FFMCFine Fuel Moisture Code미세 연료 수분 코드. 기상 변화에 민감.
FWIFire Weather Index종합 산불 기상 지수. 산불 강도 지표.

2.1.2 포르투갈 산불 데이터셋 (Forest Fires)

  • 장소: Montesinho Natural Park (517건).
  • 특징: 피해 면적(`area`)이 0에 치우친 왜곡된 분포 (데이터 불균형).
  • 전처리 핵심: 로그 변환(Log Transformation) 적용 (`ln(x+1)`) 필수.

import numpy as np

# 피해 면적의 왜곡된 분포 보정
df['log_area'] = np.log1p(df['area'])

2.2 국내 공공데이터 활용

  • 산림청 데이터: 발생 원인('기타', '입산자 실화'), 피해 면적 등. 인코딩(`cp949`), 날짜 변환 전처리 필요.
  • 기상청 ASOS 데이터: 분/시간 단위 상세 기상 정보.

2.3 데이터 정제 및 이상치 처리

2.3.1 결측치 (Missing Value)

단순 평균 대체보다는 시계열 특성을 고려한 ffill(Forward Fill)이나 선형 보간법 권장.

2.3.2 이상치 (Outlier) - IQR 기법


# IQR을 이용한 이상치 탐지 예시
Q1 = df['Ws'].quantile(0.25)
Q3 = df['Ws'].quantile(0.75)
IQR = Q3 - Q1
outliers = df[(df['Ws'] < (Q1 - 1.5 * IQR)) | (df['Ws'] > (Q3 + 1.5 * IQR))]

print(f"이상치(강풍 등) 개수: {len(outliers)}")

무조건 삭제하지 않고, '대형 산불'과의 연관성을 검토해야 함.

제4장: 머신러닝 기반
산불 예측 모델링

3일차 오전: 예측 모델 구축 및 평가

4.1 회귀 분석: 피해 면적 예측

  • 알고리즘: 선형 회귀(Baseline) → 결정 트리 → SVM (비선형 관계 학습).
  • 평가 지표:
    • RMSE: 큰 오차에 민감.
    • MAD: 이상치가 많은 산불 데이터에서 더 강건한 지표.
  • 전략: Train/Test 분할 (7:3), K-Fold 교차 검증으로 과적합 방지.

4.2 분류 분석: 발생 위험도 예측

  • 목표: Fire vs Not Fire 판별.
  • 알고리즘: 로지스틱 회귀, 랜덤 포레스트(Random Forest).
  • 랜덤 포레스트 장점: 높은 정확도 + 변수 중요도(Feature Importance) 산출 가능 (어떤 기상 요인이 핵심인지 해석).
  • 평가 지표: 재현율(Recall) 중요 (실제 산불을 놓치지 않는 것이 핵심).

4.3 딥러닝과 생성형 AI 동향

  • 딥러닝: TensorFlow/Keras 활용 다층 퍼셉트론(MLP). 위성/CCTV 이미지 분석의 기초.
  • WildfireGPT & RAG: LLM을 산불 도메인에 특화. 문헌 검색 및 시나리오 예측에 검색 증강 생성(RAG) 기술 활용.

5.1 프롬프트 엔지니어링 (Prompt Engineering)

AI에게 정확한 답변을 얻기 위한 질문 설계 기술.

  • 페르소나(Persona): "당신은 20년 경력의 산림 재난 데이터 분석가입니다."
  • 맥락(Context): "알제리 데이터를 사용 중이며, Pandas로 결측치 처리 후 시각화하려 한다."
  • 단계적 추론(Chain of Thought): "데이터 로드 → 전처리 → 시각화 순서로 코드를 설명해 달라."

5.2 AI 기반 코딩 및 디버깅

  • 에러 해결: 에러 메시지를 복사해 붙여넣고 "초보자도 알기 쉽게 원인과 해결책 설명해줘" 요청.
  • 코드 최적화: "이 코드를 더 Pythonic하고 효율적으로 개선해줘."
  • 데이터 해석: 통계 결과(상관계수 등)를 제공하고 "산불학적 의미를 해석해줘" 요청하여 논문 초안 작성 지원.
